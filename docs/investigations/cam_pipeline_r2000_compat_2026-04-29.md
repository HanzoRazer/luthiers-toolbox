**Root Cause:** The pipeline only processes LWPOLYLINE and POLYLINE entities (`mvp_router.py:83-96`). R12 files using LINE entities are not supported.

### Gap Identification

**Gap Location:** The gap is NOT in the CAM pipeline itself. The pipeline correctly processes closed LWPOLYLINE entities.

**Actual Gap:** The upstream DXF conversion process that created the test files did not assemble LINE segments into closed polylines. Each LINE was converted to a separate 2-point LWPOLYLINE.

**Evidence:**
- Cuatro file layers with 2-point entities: PICKGUARD (8), SMALL_FEATURE (291), BRIDGE_ROUTE (6), etc.
- All three sanity test files have 100% 2-point entities
- Only TEST_CLOSED_RECT (4 pts) and TEST_CLOSED_BODY (12 pts) were manually constructed as proper closed shapes

## Summary

### What Works

1. **R2000 (AC1015) format parsing:** ezdxf reads R2000 DXF files without issue
2. **Closed LWPOLYLINE processing:** The pipeline correctly extracts and toolpaths closed LWPOLYLINE entities with ≥3 points
3. **G-code output:** Syntactically valid GRBL G-code with proper structure
4. **Coordinate fidelity:** Output stays within source geometry bounds

### What Doesn't Work

1. **R12 LINE entities:** The pipeline does not process LINE entities – only LWPOLYLINE and POLYLINE
2. **2-point LWPOLYLINEs:** Entities converted from LINEs without loop assembly are filtered out
3. **Production-ready spindle control:** Missing S-word and M3 for spindle start

### Root Cause of Test File Failures

The three sanity test files (Flying V, Firebird, Double Neck) and the converted portions of the cuatro test file contain only 2-point LWPOLYLINEs. This is because the LINE-to-LWPOLYLINE conversion process did not assemble connected LINE segments into closed polylines.

**The CAM pipeline itself is not the gap.** The gap is in the upstream DXF conversion that produced these test files.

### Recommendation

To validate R2000 LWPOLYLINE CAM compatibility at scale, new test files are needed that contain:
1. Properly assembled closed LWPOLYLINEs (not 2-point line segments)
2. Multi-point closed contours representing actual guitar body shapes

The TEST_CLOSED_RECT and TEST_CLOSED_BODY entities in the cuatro test file demonstrate that the pipeline works correctly when given properly-formed input. The 487-entity test file itself is not a valid production-scale test because 485 of those entities are unconverted line segments.

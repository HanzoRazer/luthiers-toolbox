## 1. Has This Topology-Loss Problem Already Been Diagnosed?

**YES — extensively documented.**

### Key Finding: Loop Assembly Is Missing

From `docs/investigations/cam_ready_loop_assembly_2026-04-29.md`:

```
Gap Location: The gap is NOT in the CAM pipeline itself.
The pipeline correctly processes closed LWPOLYLINE entities.

Actual Gap: The upstream DXF conversion process did not assemble
LINE segments into closed polylines. Each LINE was converted
to a separate 2-point LWPOLYLINE.

Loop assembly is missing.
```

From `docs/investigations/cam_pipeline_r2000_compat_2026-04-29.md`:

```
Root Cause: These files were generated by converting LINE entities
to LWPOLYLINEs without assembling connected segments into closed loops.
Each LINE became a standalone 2-point LWPOLYLINE.
```

**Implication for Current Audit:**  
The BodyEvidence bridge is consuming the same LINE-based DXF output that was already known to lack loop assembly. The "slab_body collapse" is likely caused by the bridge treating disconnected LINE entities as isolated contour segments, destroying topology semantics.

## 3. Is There a Known-Good Way to Interpret edge_to_dxf Output?

**YES — documented in `EDGE_TO_DXF_REFINED_METHODOLOGY.md`.**

### Key Facts

| Aspect | Value |
|--------|-------|
| Format | R12 (AC1009) — LINE entities only |
| Entity structure | Each contour point → LINE segment to next point |
| Contours closed | Last point → first point (implicit) |
| Simplification | NONE (CHAIN_APPROX_NONE) |
| Layer | "EDGES" |

### Critical Insight

```
EdgeToDXF produces LINE entities (not LWPOLYLINE).
Contours are closed implicitly — last point connects to first.
But each LINE is a separate 2-point entity.
Downstream consumers must reconstruct loops from endpoint proximity.
```

**Implication:**  
The vectorizer output is topologically correct (closed contours) but structurally fragmented (disconnected LINE entities). The BodyEvidence bridge must chain LINE entities by endpoints to recover contour connectivity — it cannot treat each LINE as an independent segment.

## 5. Did Prior Audits Identify LINE-Only DXF as Intentional?

**YES — for R12 compatibility.**

From `CLAUDE.md`:

```
Free tier output: R12 (AC1009)
- LINE entities only via dxf_compat.add_polyline(version='R12')
- Maximum legacy compatibility for hobbyist users

Paid tier output: R2000 (AC1015)
- LWPOLYLINE entities
- Multi-point closed contours, suitable for CAM workflows
```

**Implication:**  
The LINE-only structure is intentional for R12 compatibility. Topology recovery must happen at consumption time (in the BodyEvidence bridge), not at generation time.

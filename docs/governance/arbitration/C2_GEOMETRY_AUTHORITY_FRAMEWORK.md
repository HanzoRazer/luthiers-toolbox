## 3. Geometry Semantic Layers

### Layer Definitions

| Layer | Meaning | Authority Scope |
|-------|---------|-----------------|
| **authoritative_geometry** | Canonical dimensional truth | Defines what geometry IS |
| **operational_geometry** | Runtime-consumable geometry | Enables execution |
| **derived_geometry** | Computed/interpreted geometry | Produces interpretations |
| **presentation_geometry** | Rendering/view geometry | Enables visualization |
| **serialized_geometry** | Export representation | Emits format artifacts |
| **morphology_geometry** | Classification-linked geometry | Enables shape reasoning |
| **sandbox_geometry** | Pre-governance geometry | Experimental semantic space |

### Layer Characteristics

#### authoritative_geometry

```
Purpose: Define canonical geometry truth
Authority: Owns dimensional semantics
Mutation: Creates geometry meaning
Consumers: All other layers
Examples: Body outline definitions, canonical dimensions
Risk: Authority source ambiguity
```

#### operational_geometry

```
Purpose: Enable runtime execution
Authority: Consumes authoritative, transforms for execution
Mutation: Adapts representation, not meaning
Consumers: CAM, Topology Builder, execution engines
Examples: RuntimeResultBase geometry references
Risk: Implicit authority escalation
```

#### derived_geometry

```
Purpose: Produce computed interpretations
Authority: NONE — interpretation only
Mutation: Creates derivative values
Consumers: Presentation, analysis systems
Examples: Helmholtz estimates from aperture geometry
Risk: Derived values treated as authoritative
```

#### presentation_geometry

```
Purpose: Enable visualization and rendering
Authority: NONE — display only
Mutation: Transforms for display, not meaning
Consumers: UI components, preview systems
Examples: SVG paths, viewport transforms
Risk: Presentation becoming de facto truth
```

#### serialized_geometry

```
Purpose: Emit format artifacts
Authority: NONE — representation only
Mutation: Encodes, does not define
Consumers: External tools, file systems
Examples: DXF contours, JSON exports
Risk: Serialization format acquiring semantic authority
```

#### morphology_geometry

```
Purpose: Enable shape classification
Authority: Classification-scoped only
Mutation: Classifies, does not define shape
Consumers: Morphology systems, corpus builders
Examples: IBG zone radii, bout classifications
Risk: Classification becoming shape authority
```

#### sandbox_geometry

```
Purpose: Experimental semantic development
Authority: NONE — pre-governance
Mutation: Explores semantic possibilities
Consumers: Development workflows only
Examples: IBG body grid vocabulary
Risk: Sandbox semantics leaking to production
```

## 4. Ownership Categories

### Role Definitions

| Role | Meaning | Authority Rights |
|------|---------|------------------|
| **authoritative_owner** | Defines geometry truth | Create, mutate, ratify meaning |
| **constrained_transformer** | Adapts geometry for purpose | Transform representation, not meaning |
| **serializer** | Emits representation | Encode only, no semantic mutation |
| **observational_consumer** | Reads geometry | Read only, no mutation |
| **operational_consumer** | Runtime consumption | Consume for execution, no definition |
| **sandbox_incubator** | Experimental producer | Create experimental semantics only |
| **derived_producer** | Computes interpretations | Derive values, not authority |

### Role Constraints

#### authoritative_owner

```
MAY:
- Define geometry meaning
- Establish canonical dimensions
- Ratify semantic changes
- Authorize derived values

MUST NOT:
- Exist without constitutional registration
- Delegate authority without documentation
- Allow silent meaning mutation
```

#### constrained_transformer

```
MAY:
- Adapt geometry for specific purpose
- Transform coordinate systems
- Optimize representation for target

MUST NOT:
- Redefine geometry meaning
- Assert authority over transformed output
- Create semantic binding from transformation
```

#### serializer

```
MAY:
- Encode geometry to format
- Apply format-specific conventions
- Add format metadata

MUST NOT:
- Define geometry truth via format
- Create semantic authority from serialization
- Allow format to imply ownership
```

#### observational_consumer

```
MAY:
- Read geometry values
- Derive observations
- Store snapshots

MUST NOT:
- Modify geometry meaning
- Assert ownership via consumption
- Escalate observation to authority
```

## 5. Propagation Constraints

### Authority Movement Rules

```
RULE 1: Authority does not flow downstream without explicit transfer
RULE 2: Derived geometry does not become authoritative by usage
RULE 3: Serialization does not create authority
RULE 4: Consumption does not imply ownership
RULE 5: Sandbox geometry does not escape containment without governance
```

### Propagation Boundaries

| From Layer | To Layer | Permitted? | Constraint |
|------------|----------|------------|------------|
| authoritative | operational | YES | Explicit consumption contract |
| authoritative | derived | YES | Must carry provenance |
| authoritative | presentation | YES | Must not mutate |
| authoritative | serialized | YES | Must not define |
| operational | presentation | YES | No authority escalation |
| derived | presentation | YES | Must carry assumptions |
| derived | authoritative | NO | Derived cannot escalate |
| serialized | authoritative | NO | Format is not truth |
| sandbox | authoritative | NO | Requires governance gate |
| sandbox | production | NO | Containment boundary |

### Escalation Prevention

```
ANTI-PATTERN 1: Derived geometry treated as measured
ANTI-PATTERN 2: Serialized format defining geometry semantics
ANTI-PATTERN 3: Presentation values becoming canonical
ANTI-PATTERN 4: Sandbox vocabulary escaping to production
ANTI-PATTERN 5: Operational geometry silently becoming source
```

## 7. Serialization Boundaries

### Core Principle

```
Serializers emit representations.
Serializers do not define geometry truth.
Format authority is format-scoped, not geometry-scoped.
```

### Serialization Rules

| Rule | Description |
|------|-------------|
| No semantic creation | Format cannot create geometry meaning |
| Representation only | Output is encoding, not definition |
| Format metadata | May include format-specific data |
| Authority passthrough | Serializer passes authority, does not add |

### Export Domain Classification

```
DXF Export → serialized_geometry (no authority)
JSON Export → serialized_geometry (no authority)
STEP Export → serialized_geometry (no authority)
CAD Translation → serialized_geometry (no authority)
```

## 4. Derived Semantic System Definition

### What Is a Derived Semantic System?

```
A derived semantic system is any operational system that:
- Produces semantic outputs
- Constrains semantic paths
- Influences semantic interpretation
- Transforms semantic representations
- Persists semantic state

WITHOUT:
- Owning semantic definitions
- Holding ratification authority
- Creating governance
```

### Constitutional Classification

| Property | Derived Semantic System | Semantic Authority |
|----------|------------------------|-------------------|
| Produces semantic output | YES | YES |
| Influences semantic paths | YES | YES |
| Defines semantic meaning | NO | YES |
| Holds ratification power | NO | YES |
| Creates governance | NO | YES |
| Provenance type | PROV_DERIVATION | PROV_AUTHORITY |
| Authority-state | derived / operational | canonical |

### Key Distinction

```
Derived semantic systems operate ON semantics.
Semantic authorities operate AS semantics.
```

### 5.3 Category Definitions

#### Evaluator

```
Purpose: Assess semantic quality or state
Function: Produces evaluation results about semantics
Risk: Evaluation results treated as semantic truth
Example: Continuity evaluator assessing G0/G1/G2 quality
Constitutional requirement: PROV_DERIVATION, non-authority marker
```

#### Validator

```
Purpose: Gate semantic transitions or operations
Function: Approves or rejects semantic operations
Risk: Validation criteria become semantic law
Example: Topology validator checking surface relationships
Constitutional requirement: Constraint provenance, scope limitation
```

#### Analyzer

```
Purpose: Derive insights from semantic data
Function: Produces derived semantic observations
Risk: Derived observations treated as authoritative
Example: Geometry analyzer computing derived measurements
Constitutional requirement: Derivation chain visibility, epistemic markers
```

#### Translator

```
Purpose: Transform semantic representations
Function: Converts between semantic formats
Risk: Transformation introduces semantic mutation
Example: Export translator converting internal to external format
Constitutional requirement: Transformation provenance, quarantine enforcement
```

#### Cache

```
Purpose: Persist semantic state for performance
Function: Stores and retrieves semantic data
Risk: Cached values harden into authority
Example: Derived geometry cache storing computed values
Constitutional requirement: Source provenance persistence, authority-state preservation
```

#### Serializer

```
Purpose: Encode semantics for storage or transmission
Function: Converts semantic state to persistent form
Risk: Serialization amplifies semantic propagation
Example: DXF serializer encoding geometry for export
Constitutional requirement: Export quarantine, propagation documentation
```

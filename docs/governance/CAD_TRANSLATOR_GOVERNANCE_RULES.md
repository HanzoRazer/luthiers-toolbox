# CAD Translator Governance Rules

**Date:** 2026-05-14  
**Sprint:** MRP-5A  
**Status:** ACTIVE

---

## Purpose

Defines governance rules specifically for CAD-grade translators (STEP, IGES, etc.) that extend the core TRANSLATOR_LAYER_RULES.md with CAD-specific constraints.

---

## Core Principle

**CAD translators may CONSTRUCT topology from approved geometry, but may NOT REINTERPRET approved geometry authority.**

This means:
- Translators CAN build 3D solids from 2D profiles
- Translators CAN derive faces and edges from approved contours
- Translators CANNOT modify the approved 2D coordinates
- Translators CANNOT add geometry not derivable from Export Object

---

## Authority Boundaries

### Export Object Authority (Upstream)

The Export Object is the canonical source for:

| Authority | Owned By | CAD Translator Role |
|-----------|----------|---------------------|
| Profile coordinates (x, y) | Export Object | READ ONLY |
| Contour closure | Export Object validation | CONSUME |
| Winding direction | Export Object | CONSUME |
| Coordinate system | Export Object | CONSUME |
| Manufacturing intent | Export Object | CONSUME |
| Morphology (side_heights) | IBG extension | CONSUME |

### CAD Translator Authority

CAD translators are authorized to:

| Authority | Translator Owns | Constraints |
|-----------|-----------------|-------------|
| Topology construction | YES | From approved geometry only |
| Face generation | YES | Derived from profile + thickness |
| Edge derivation | YES | From face intersections |
| Solid body creation | YES | Watertight from approved profiles |
| Surface generation | YES | Ruled/swept from approved curves |
| BREP representation | YES | Faithful to profile geometry |

### Forbidden CAD Translator Behaviors

CAD translators are NOT authorized to:

| Forbidden Action | Reason |
|------------------|--------|
| Modify profile coordinates | Approved geometry is immutable |
| Invent new geometry | No geometry authority |
| Change winding direction | Topology semantics preserved |
| Add constraints not in Export Object | No parametric authority |
| Generate toolpaths or G-code | Manufacturing authority separate |
| Merge entities without approval | Entity identity preserved |
| Infer missing morphology | Morphology authority is IBG |

## CAD Translator Responsibilities

### 1. Geometry Fidelity

```
Input profile points  →  Output STEP vertices
Must satisfy: |input[i] - output[i]| < tolerance (default 0.001 mm)
```

Profile coordinates in the STEP output must match Export Object coordinates within manufacturing tolerance.

### 2. Topology Construction

CAD translators construct topology through deterministic algorithms:

```
Profile (2D) + Thickness (scalar) → Solid (3D)

Construction method:
1. Create wire from profile points
2. Extrude wire by thickness in z_axis direction
3. Cap top and bottom faces
4. Validate watertight
```

The construction algorithm must be documented and deterministic.

### 3. Morphology Consumption

When IBG morphology extension is present:

```python
if export_object.extensions.ibg_morphology.side_heights_mm:
    # Variable extrusion path
    heights = export_object.extensions.ibg_morphology.side_heights_mm
    assert len(heights) == len(profile_points)
    # Pair each profile point with its height
else:
    # Constant extrusion path
    thickness = export_object.extensions.thickness_mm  # or intent-derived
```

Translators CONSUME morphology, they do not GENERATE it.

### 4. Gate Enforcement

| Gate | CAD Translator Action |
|------|----------------------|
| GREEN | Translation proceeds |
| YELLOW | Translation proceeds with topology warnings |
| RED | Translation blocked — no STEP output |

**Red gate rejection is mandatory.** CAD translators must refuse to generate STEP files from red-gated Export Objects.

### 5. Provenance Requirements

CAD translators MUST embed provenance in STEP header:

```
/* STEP file generated by Luthiers Toolbox */
/* export_id: EXP-BODY-20260514-abc123 */
/* translator_id: body_outline_step_ap203 */
/* translator_version: 1.0.0 */
/* translated_at: 2026-05-14T12:00:00Z */
```

STEP files must be traceable to their source Export Object.

## Topology Construction Rules

### Simple Extrusion (Flat Body)

```
Profile: closed_contour from entities[0]
Direction: coordinate_system.z_axis (positive = toward top face)
Depth: thickness_mm (from extensions or derived)

Resulting topology:
- 1 top face (planar, z = thickness)
- 1 bottom face (planar, z = 0)
- N side faces (one per profile edge)
- M void faces (if voids present)
```

### Variable Extrusion (Acoustic Body)

```
Profile: closed_contour from entities[0]
Heights: side_heights_mm from ibg_morphology
Direction: coordinate_system.z_axis

Resulting topology:
- 1 top face (may be non-planar if arch-top)
- 1 bottom face (planar, z = 0)
- N side faces (ruled surfaces with variable height)
```

### Void Handling

```
For each void in entities where role != "outer":
- Create profile wire from void.points
- Extrude through body thickness
- Subtract from solid (CSG difference)
- Validate resulting topology
```

## Validation Requirements

### Pre-Translation Checks

| Check | Required | Gate Impact |
|-------|----------|-------------|
| Contour closed | Yes | RED if failed |
| Minimum points | Yes | RED if < 3 |
| Valid coordinate system | Yes | RED if missing |
| Thickness available | Yes | RED if not derivable |

### Post-Translation Checks

| Check | Required | Action |
|-------|----------|--------|
| Watertight topology | Yes | Fail if not watertight |
| Face count matches expectation | Yes | Warn if mismatch |
| Volume positive | Yes | Fail if zero/negative |
| No self-intersection | Yes | Fail if detected |

## Error Handling

### Translator Error Codes (CAD-specific)

| Code | Meaning | Action |
|------|---------|--------|
| `TOPOLOGY_CONSTRUCTION_FAILED` | Could not build valid solid | Return error, no output |
| `NOT_WATERTIGHT` | Resulting solid has gaps | Return error, no output |
| `THICKNESS_MISSING` | No thickness value available | Return error with message |
| `MORPHOLOGY_INCONSISTENT` | side_heights count != points count | Return error |
| `COMPLEX_MORPHOLOGY_UNSUPPORTED` | Translator cannot handle arch-top | Return error with classification |

### Graceful Degradation

```python
if complex_morphology_detected and not supports_complex:
    return TranslatorResult(
        success=False,
        error_code="COMPLEX_MORPHOLOGY_UNSUPPORTED",
        error_message="This translator only supports flat-body geometry. "
                      "Use body_acoustic_step_ap203 for variable profiles.",
        suggested_translator="body_acoustic_step_ap203"
    )
```

## Determinism Requirements

CAD translators MUST produce deterministic output:

```
Same Export Object + Same options → Same STEP file (excluding timestamps)
```

Determinism requirements:
- Vertex numbering must be stable
- Face ordering must be deterministic
- Edge references must be consistent
- No random IDs or UUIDs

## STEP-Specific Constraints

### Supported Application Protocol

| Protocol | Status | Use Case |
|----------|--------|----------|
| AP203 | TARGET | Configuration-controlled 3D design |
| AP214 | FUTURE | Extended automotive features |
| AP242 | FUTURE | Model-based definition |

**MRP-5A targets AP203 only.**

### Entity Constraints

| STEP Entity | Allowed | Notes |
|-------------|---------|-------|
| CARTESIAN_POINT | Yes | From Export Object coordinates |
| POLYLINE | Yes | From profile points |
| PLANE | Yes | Top/bottom faces |
| ADVANCED_FACE | Yes | All faces |
| CLOSED_SHELL | Yes | Watertight body |
| MANIFOLD_SOLID_BREP | Yes | Target entity type |

### Unit Handling

```
Export Object units: "mm" (always)
STEP units: millimetres (LENGTH_UNIT)
Conversion: None required (mm → mm)
```

## Registration Requirements

CAD translators MUST register with TranslatorRegistry:

```python
from app.cam.translators.base import (
    TranslatorCapabilities,
    TranslatorCategory,
    ExecutionState,
)

BODY_STEP_CAPABILITIES = TranslatorCapabilities(
    translator_id="body_outline_step_ap203",
    translator_name="Body Outline STEP AP203 Translator",
    translator_version="1.0.0",
    target_format="step",
    format_version="AP203",
    category=TranslatorCategory.SERIALIZATION,
    execution_state=ExecutionState.GOVERNED_EXECUTION,
    execution_supported=True,
    artifact_generation_supported=True,
    machine_output_supported=False,  # STEP is interchange, not machine code
    provenance_supported=True,
    deterministic=True,
    supported_operations=["body_profiling"],
    supported_geometry_types=["closed_contour"],
    description="STEP AP203 solid body translator",
)
```

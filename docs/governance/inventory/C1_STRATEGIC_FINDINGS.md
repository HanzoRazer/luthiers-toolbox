## 3. Cross-Domain Governance Patterns

### Pattern A: Consumer-Without-Authority

**Discovery:** Acoustics/Observational inventory

**Finding:**
```
Acoustics demonstrates that a subsystem may consume geometry-derived 
information without asserting geometry authority ownership.
```

**Evidence:**
- `ApertureGeometryLike` — consumer interface, explicitly decoupled
- `MeasurementArchiveGeometrySummary` — snapshot, not authority
- `apertureGeometry` field — "reference, not merged"
- Code comment: "Geometry remains separate (ApertureGeometry is not merged into this)"

**Strategic Implication:**
This pattern may become a governance reference model for:
- Translators
- Export systems
- Runtime visualization
- Topology consumption layers

**Constitutional Significance:**
```
Consumers may derive observations from a domain
without acquiring authority over that domain.
```

### Pattern B: Bounded Authority Declaration

**Discovery:** Acoustics/Observational inventory

**Finding:**
All acoustic authorities explicitly declare non-goals:
- MeasurementArchive: "does NOT store calibration outputs, predictions, or recommendations"
- DiagnosticSnapshot: "Observational only — does NOT persist, calibrate, or predict"
- AcousticState: "descriptive model, not a solver"

**Strategic Implication:**
Explicit non-goal documentation prevents authority creep more effectively than enforcement alone.

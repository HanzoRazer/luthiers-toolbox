# Strategy Package Manifest

## Overview

A strategy package manifest bundles validated CAM Assist artifacts into a portable, reviewable package:

- Strategy JSON (validated against A2 schema)
- Review packet (generated by A3)
- Optional geometry file references
- Execution authority declarations
- Provenance metadata

**All artifact paths in a manifest are resolved relative to the manifest file location.**

## Non-Execution Declaration

Strategy package manifests **explicitly deny execution authority**:

```json
"authority": {
  "non_execution_declaration": true,
  "execution_authority_claim": false,
  "requires_human_review": true
}
```

Any manifest that does not have these exact values will fail validation.

## Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `manifest_version` | string | Semantic version (e.g., "1.0.0") |
| `package_type` | string | Must be "cam_assist_strategy_package" |
| `operation_type` | string | Type of operation (e.g., "fret_slot_strategy") |
| `strategy_file` | string | Path to strategy JSON, relative to manifest |
| `review_packet_file` | string | Path to review packet, relative to manifest |
| `authority` | object | Execution authority declarations |
| `created_at` | string | ISO-8601 timestamp |

## Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `source_geometry_files` | array | Paths to geometry files (DXF, etc.) |
| `cam_assist_version` | string | Version that created the package |
| `provenance` | object | Origin and derivation metadata |
| `notes` | string | Additional notes |

## Example Manifest

```json
{
  "manifest_version": "1.0.0",
  "package_type": "cam_assist_strategy_package",
  "operation_type": "fret_slot_strategy",

  "strategy_file": "fret_slot_strategy.json",
  "review_packet_file": "fret_slot_strategy_review_packet.md",
  "source_geometry_files": [],

  "created_at": "2026-05-21T16:00:00Z",
  "cam_assist_version": "0.4.0",

  "authority": {
    "non_execution_declaration": true,
    "execution_authority_claim": false,
    "requires_human_review": true
  },

  "provenance": {
    "source_spec_id": "sample-stratocaster-spec",
    "created_by": "cam-assist-blueprint",
    "derivation_notes": "Example manifest for fret slot strategy"
  }
}
```

## Validation

```bash
python scripts/validate_manifest.py examples/valid/fret_slot_strategy_manifest.json
```

### Validation Rules

**Must fail if:**
- `non_execution_declaration` is not `true`
- `execution_authority_claim` is not `false`
- `requires_human_review` is not `true`
- `strategy_file` is missing or file doesn't exist
- `review_packet_file` is missing or file doesn't exist
- `package_type` is not "cam_assist_strategy_package"

**Should warn if:**
- `source_geometry_files` is empty
- `provenance.created_by` is empty
- `provenance.derivation_notes` is empty
- `cam_assist_version` is missing

### Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Manifest is valid |
| 1 | Validation failed |
| 2 | File/read/schema error |

## Path Resolution

All paths in the manifest are relative to the manifest file's directory:

```
my_package/
├── manifest.json           # The manifest
├── strategy.json           # strategy_file: "strategy.json"
├── review_packet.md        # review_packet_file: "review_packet.md"
└── geometry/
    └── slots.dxf           # source_geometry_files: ["geometry/slots.dxf"]
```

This makes packages portable — they can be moved or archived together.

## Integration with A1-A3

| Artifact | Source | Manifest Reference |
|----------|--------|-------------------|
| Strategy JSON | A2 validated | `strategy_file` |
| Review Packet | A3 generated | `review_packet_file` |
| Geometry | A0 sample or external | `source_geometry_files` |

## Use Cases

1. **Archival**: Bundle a complete strategy package for project records
2. **Review handoff**: Send package to another luthier for review
3. **Downstream CAM**: Provide validated inputs to CAM software
4. **Build documentation**: Attach to instrument build records

## What This Is Not

Strategy package manifests do **not**:
- Generate G-code
- Authorize machine execution
- Replace CAM software
- Bypass human review requirements

They are a **documentation and handoff format**, not an execution format.

## 1. Scope

C2 is **constitutional semantic arbitration** inside a federated governance system.

### 1.1 What C2 Is

| Scope Item | Description |
|------------|-------------|
| Arbitration framework | Constitutional process for semantic reconciliation |
| Ownership topology | Authority decomposition across domains |
| Namespace decomposition | Candidate distinctions for semantic surfaces |
| Lifecycle axis modeling | Multi-dimensional lifecycle preservation |
| Provenance decomposition | Epistemic category separation |
| Geometry authority packet | C2-A: first concrete arbitration |
| IBG federation boundary | Planning only (execution deferred) |

### 1.2 Constitutional Principle

```
Constitutional reconciliation precedes implementation reconciliation.
```

The repository is now coordinating **federated semantic legitimacy** across runtime, topology, provenance, export, governance, and sandbox semantic systems.

### 1.3 Preservation Principle

```
Preserving semantic distinction is more important than
simplifying semantic structure.
```

Until decomposition stabilizes, propagation is understood, and constitutional boundaries become trustworthy.

C2 explicitly excludes the following activities:

| Non-Goal | Reason |
|----------|--------|
| Registry migrations | Requires ratified arbitration pathways first |
| Code rewrites | Implementation follows constitutional stabilization |
| CI blocking changes | Enforcement follows arbitration consensus |

| Auto-reconciliation tooling | Human auditability required |
| IBG federation execution | Planning only during C2 |
| Translator refactoring | Export authority review only |
| Runtime dispatcher changes | Propagation analysis only |

### 2.1 Critical Distinction

C2 is NOT:
```
Ontology cleanup
```

C2 IS:
```
Constitutional semantic arbitration
inside a federated governance system
```

## 5. Arbitration Packet Protocol

### 5.1 Required Packet Sections

| # | Section | Purpose |
|---|---------|---------|
| 1 | Packet Identity | ID, phase, date, status |
| 2 | Constitutional Scope | What semantic surface is arbitrated |
| 3 | C1 Evidence Inputs | Which C1 findings inform this packet |
| 4 | Collision Description | Specific collisions being addressed |
| 5 | Authority Claimants | Systems claiming semantic authority |
| 6 | Semantic Decomposition | Proposed distinction preservation |
| 7 | Ownership Topology | Who owns what after arbitration |
| 8 | Namespace Analysis | Term/concept namespace evaluation |
| 9 | Propagation Analysis | How semantics flow through runtime (T2) |
| 10 | Observational Risks | Provenance/consumer threats (T4) |
| 11 | Runtime Risks | Execution boundary threats (T2) |
| 12 | Export/Serialization Risks | Authority leakage threats (T5) |
| 13 | Federation Blockers | What prevents immediate federation |
| 14 | Unresolved Ambiguities | Explicit ambiguity preservation |
| 15 | Escalation Triggers | Conditions requiring Tier escalation |
| 16 | Candidate Decomposition Paths | Proposed resolution options |
| 17 | Dissent Preservation | Explicit disagreement documentation |
| 18 | Human Arbitration Requirements | What requires human decision |
| 19 | Ratification Preconditions | What must be true before ratification |
| 20 | Constitutional Status | Current packet state |

### 5.2 Sign-Off Protocol

All affected terminals must review packets:

| Terminal | Review Responsibility |
|----------|----------------------|
| T1 | Constitutional governance, process integrity |
| T2 | Runtime propagation, operational escalation |
| T3 | Semantic decomposition, namespace arbitration |
| T4 | Provenance/observational discipline |
| T5 | Export/serialization authority boundaries |

**Dissent must remain visible.** Packets do not require unanimous agreement, but disagreement must be explicitly preserved in Section 17.

### 5.3 Escalation Tiers

| Tier | Meaning | Action |
|------|---------|--------|
| 0 | Local clarification | Terminal resolves internally |
| 1 | Namespace ambiguity | Cross-terminal discussion |
| 2 | Cross-domain conflict | T1 coordination required |
| 3 | Authority collision | Formal arbitration packet |
| 4 | Constitutional arbitration | Human owner decision required |

### 5.4 Escalation Principle

```
Preserved ambiguity is preferred over premature semantic convergence.
```

Escalate only if decomposition fails. Never silently collapse disagreement.

**Role:** Inventories runtime impacts, validates execution/authority boundaries

**Deliverables:**

### 9.1 Phase Sequence

| Phase | Packet | Scope | Prerequisite |
|-------|--------|-------|--------------|
| C2-A | Geometry Authority Decomposition | Authority chain formalization | C1 frozen |
| C2-B | Topology Namespace Arbitration | Namespace distinction proposals | C2-A complete |

| C2-D | Provenance Decomposition | Epistemic category formalization | C2-A complete |
| C2-E | Export Authority Boundaries | Serialization non-authority | C2-A complete |

### 9.2 C2-A First

C2-A (Geometry Authority Decomposition) is the first packet because:

1. Geometry authority is the highest-pressure undeclared dependency
2. Multiple systems consume geometry without declared authority
3. IBG, topology, and CAM all depend on geometry authority clarity
4. Resolving geometry authority unblocks subsequent arbitration

### 9.3 Parallelization

After C2-A completes:
- C2-B, C2-D, C2-E may proceed in parallel
- C2-C requires C2-B completion (IBG depends on topology clarity)
